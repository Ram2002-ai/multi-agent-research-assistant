# Contributing

Thanks for improving ResearchOS.

1. Create a focused branch.
2. Keep the `/research` response and five-agent context flow backward compatible.
3. Add tests for behavior changes.
4. Run `pytest`, `ruff check`, and `npm run build`.
5. Never commit provider keys, user research, generated logs, or production data.
6. Explain architecture changes and migration impact in the pull request.

Use conventional commit subjects where practical: `feat:`, `fix:`, `docs:`,
`test:`, `refactor:`, or `chore:`.
